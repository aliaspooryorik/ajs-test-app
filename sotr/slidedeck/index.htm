What is AngularJS?

Superheroic JavaScript MVW Framework
http://angularjs.org/
Model, View, Whatever
  • The slogan of AngularJS is Superheroic JavaScript MVW Framework
  • MVW stands for Model, View, Whatever
  • declarative programming for UI
  • Imperative programming for model
  • AngularJS is a client-side JavaScript framework that lets you extend HTML vocabulary for your application. The resulting environment is extraordinarily expressive, readable, and quick to develop.

What is AngularJS?

https://twitter.com/aliaspooryorik/status/230405938597728256
  • Mean; "enhanced" HTML
  • Mean; templates written in HTML (not in JavaScript)
  • Mean; Extend HTML with new tags
  • Looks a bit like ColdFusion tags!
  • HTML was designed for static websites. Now the web is a much richer place and you're probably using something like jQuery to manipulate your DOM and listen to events such as on click etc. Angular if you like extends HTML and gives it behaviour. It makes use of event listeners but these are abstracted away so you just get automatic two way data binding.....
  • Have you ever wanted to have tabs on your website? With HTML you have a bunch of divs and CSS. With angular you can create an new HTML tag called "tabs" if you wanted.

Model-View Glue

<ul ng-app ng-controller="MyCtrl">
  <li ng-repeat="who in speakers">
	{{who.surname}}, {{who.forename}}</li>
</ul>

<script>
function MyCtrl ($scope) {
  $scope.speakers = [
    {forename:'John', surname:'Whish'},
    {forename:'Matt', surname:'Gifford'},
    {forename:'Rob', surname:'Dudley'}
  ];
}
</script>

↓ controller binding

model

↓ DI

Having data in the view is bad. Want it in the model I've removed the html / body tags for clarity
Note the ng-controller directive
Note $scope is automatically injected into the controller (DI)
Note $scope is automatically injected into the view
speakers in the view is the same as $scope.speakers in model
